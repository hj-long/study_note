## 权限管理


用户

角色：岗位，权限不同

按钮级别的权限管理：

1. 页面上所有的按钮 和 超链接 都可以控制：

    - ['system:role:add', 'system:role:delete']: 系统角色添加权限
    删除权限

    - 在 src/directives目录下新建自定义指令文件，使用自定义指令完成按钮级别的指令问题

    ```
    app.directive('指令名称', () => {
        mounted(el, binding) {
            el: 写了指令的元素节点
            binding.value: 指令的值
            流程：
                1. 从pinia中取出权限编码表，对比 指令value，是否有这个按钮权限

                2. 如果有，就显示该按钮，如果没有，找到el的父节点，然后删除本节点，达到把把无权限的按钮去除的目的
        }
    })

    ```



## mock 数据的依据

在后端还没有接口数据的时候，需要mock数据

1. 不能直接写在标签上，

2. 页面上可能变化的数据，都需要mock：也就是需要调接口的数据

这个和组件拆分依据一样的：数据有没有可能发生变化，有可能就拆分为组件，
                        没有就不要过度拆分
